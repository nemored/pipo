** Build and run
*** Rust (legacy)
#+BEGIN_SRC bash
cargo run -- path-to-config.json path-to-db.sqlite3
#+END_SRC

You can also provide the paths via environment variables:

#+BEGIN_SRC bash
CONFIG_PATH=path-to-config.json DB_PATH=path-to-db.sqlite3 cargo run
#+END_SRC

*** Go rewrite branch workflow
The Go rewrite currently focuses on CLI compatibility (argument/env parsing and usage output).

#+BEGIN_SRC bash
# Build the Go binary
go build -o pipo-go ./cmd/pipo

# Run with positional args
./pipo-go path-to-config.json path-to-db.sqlite3

# Or run with environment-variable fallback
CONFIG_PATH=path-to-config.json DB_PATH=path-to-db.sqlite3 ./pipo-go
#+END_SRC

** Protobuf generation contract
- Source proto path: =protos/Mumble.proto=
- Generated output path: =src/protos=
- Regeneration command: =cargo build=
- The generated files under =src/protos= are intentionally ignored by git and are expected to be rebuilt deterministically by =build.rs= using vendored =protoc=.

** Protobuf/CI acceptance criteria
- Two clean builds in fresh environments produce identical build behavior.
- CI fails if protobuf generator drift introduces unexpected repository changes.
- CI clean-regeneration drift checks run on the checked-out commit and only fail when build steps mutate tracked files unexpectedly (not because the PR contains committed changes).
- CI uses lockfile-locked cargo invocations (e.g. =cargo build --locked=) to avoid dependency-resolution drift between environments.
- CI is expected to pass with protobuf runtime =\>= 3.7.2=.
